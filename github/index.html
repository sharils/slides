create a repository

  1. switch to the cygwin window
  2. cd ~/projects/lab
  3. git remote add origin git@github.com:<username>/lab.git
  4. the string github.com must match the the host in your ~/.ssh/config file.
  5. git push --set-upstream origin master

git remote add ... tells git there is a remote repository call origin and it's at the spcific address

git push ... associate your local master branch with the remote master branch.

fork a repository

  1. switch to the cygwin window
  2. cd ~/projects/
  3. git clone git://github.com/<username>/software-development.git

fork a repository

  1. git config remote.origin.pushurl `git config remote.origin.url`
  2. git config remote.origin.url git://github.com/sharils/software-development.git

This tells git to always push to your own repository and fetch from the original repository. In this way, when you consider your work is done, you can send a pull request to the original repository.
